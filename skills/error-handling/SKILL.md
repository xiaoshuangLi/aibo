## 🎯 PURPOSE & SCOPE
The error-handling skill provides a comprehensive framework for detecting, recovering from, and learning from errors across all AIBO operations. This skill ensures that failures become opportunities for improvement rather than obstacles, maintaining system resilience while continuously enhancing performance through intelligent self-reflection and adaptive learning.

### When to Use This Skill
- When encountering any unexpected behavior, incorrect outputs, or system failures
- When user feedback indicates misalignment between expectations and results
- When proactive error detection identifies potential issues before they manifest
- When conducting performance retrospectives and system optimization
- When implementing new capabilities that require robust error handling protocols

### Key Principles
This skill operates on three fundamental principles: proactive error detection through multi-dimensional validation, intelligent recovery through tiered response strategies, and continuous self-improvement through systematic learning loops—ensuring that every error contributes to system evolution.

## 📋 DETAILED CAPABILITIES

### 1. **Proactive Error Detection**
- **Multi-dimensional Consistency Checks**: Logical, factual, and contextual validation
- **Anomaly Monitoring**: Real-time detection of unusual behavior patterns
- **Intent Verification**: Continuous validation of user intent understanding
- **Output Quality Assessment**: Automatic evaluation of response quality metrics
- **Cross-Reference Validation**: Verification against multiple knowledge sources

- **Tiered Error Response**: Graded responses from minor corrections to complete restarts

- **Predictive Error Prevention**: Pattern-based prediction of potential failures
- **Graceful Degradation**: Maintaining partial functionality during partial failures
- **Rollback Capabilities**: Reverting to previous stable states when necessary

### 3. **Self-Reflection & Learning Loop**
- **Performance Retrospective**: Regular analysis of system performance and outcomes
- **Error Pattern Recognition**: Identification and classification of recurring error types
- **Preventive Measure Generation**: Automatic creation of strategies to avoid future errors
- **Capability Gap Identification**: Recognition of knowledge or skill deficiencies with learning plans
- **Mental Model Calibration**: Continuous adjustment of user expectation understanding
- **Knowledge Base Updates**: Integration of learned insights into long-term memory

### 4. **User Experience Protection**
- **Transparent Communication**: Clear explanation of errors and recovery processes
- **Confidence Indicators**: Honest assessment of solution reliability and uncertainty
- **Alternative Presentations**: Multiple ways to present solutions when primary approach fails
- **Progress Preservation**: Maintaining user progress even during error recovery
- **Expectation Management**: Setting realistic expectations about system capabilities and limitations

5. Update knowledge base and mental models to prevent recurrence

## ⚠️ BEST PRACTICES & WARNINGS

### Error Handling Guidelines
- **Early Detection Priority**: Detect errors as early as possible in processing pipelines
- **Recovery Speed**: Balance thorough recovery with minimal disruption to user workflow
- **Learning Depth**: Invest appropriate analysis effort proportional to error impact
- **User Agency**: Provide users with meaningful choices during recovery processes

### Communication Principles
- **Honesty Over Perfection**: Prefer transparent acknowledgment of errors over false confidence
- **Solution Focus**: Emphasize recovery and improvement rather than dwelling on failures
- **Appropriate Detail**: Match error explanation depth to user technical expertise
- **Positive Framing**: Present errors as opportunities for collaborative improvement

### Common Pitfalls to Avoid
- **Error Hiding**: Never conceal or ignore detected errors
- **Over-Recovery**: Don't implement excessive recovery measures for minor issues
- **Blame Shifting**: Avoid attributing errors to user actions without clear evidence
- **Learning Neglect**: Don't skip the learning phase after error recovery
- **Pattern Blindness**: Recognize recurring error patterns and address root causes
