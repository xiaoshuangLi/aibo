## 🎯 PURPOSE & SCOPE
The multi-agent-collaboration skill provides a sophisticated framework for coordinating specialized AI agents to tackle complex, multi-faceted programming challenges through distributed expertise and parallel execution. This skill enables AIBO to orchestrate teams of specialized agents, each contributing their unique capabilities while maintaining coherent coordination and quality assurance.

### When to Use This Skill
- When facing complex tasks that benefit from specialized domain expertise
- When multiple subtasks can be executed independently in parallel
- When requiring diverse perspectives or cross-domain knowledge integration
- When high-stakes operations need validation and verification layers
- When optimizing performance through concurrent task execution

### Key Principles
This skill operates on three core principles: specialized role assignment based on expertise domains, parallel execution for independent tasks to maximize performance, and structured coordination protocols to ensure coherent results—transforming complex challenges into manageable, distributed workflows.

## ⚠️ BEST PRACTICES & WARNINGS

### Collaboration Guidelines
- **Independence First**: Always prioritize parallel execution of loosely coupled tasks
- **Clear Boundaries**: Define precise responsibilities and interfaces between agents
- **Communication Efficiency**: Minimize unnecessary inter-agent communication overhead
- **Quality Assurance**: Ensure validation agents review all critical outputs
- **Progress Transparency**: Maintain clear visibility into team progress and status

### Performance Considerations
- **Overhead Awareness**: Balance collaboration benefits against coordination overhead
- **Resource Constraints**: Consider available computational resources when scaling agent teams
- **Task Granularity**: Avoid creating too many micro-tasks that increase coordination costs
- **Bottleneck Identification**: Monitor for coordination bottlenecks and optimize accordingly

### Common Pitfalls to Avoid
- **Unnecessary Complexity**: Don't use multi-agent collaboration for simple, single-domain tasks
- **Poor Dependency Management**: Ensure proper handling of task interdependencies
- **Communication Overload**: Avoid excessive messaging between agents
- **Role Confusion**: Prevent overlapping or unclear agent responsibilities
- **Quality Gaps**: Don't skip validation and verification steps in complex workflows
