# 双击语音输入模式优化

## 📋 功能概述 (Feature Overview)

### 🎯 一句话描述
优化语音输入交互体验，通过双击空格键触发语音录制，提供更直观、便捷的语音输入方式。

### 💡 业务价值
- **解决的问题**: 原有的按住快捷键录音方式不够直观，用户难以掌握正确的操作时机，且与常见的双击交互模式不一致。
- **带来的价值**: 提供符合用户直觉的双击交互模式，简化语音输入操作流程，提升用户体验和使用效率。
- **目标用户**: 所有使用AIBO语音输入功能的用户，特别是希望快速、简单地进行语音交互的用户。

### 🔗 相关背景
- **相关 Issue/PR**: 基于用户反馈优化语音输入交互体验
- **设计文档**: 基于人机交互最佳实践，采用双击作为触发手势
- **依赖项**: 依赖于现有的Tencent Cloud ASR功能（feature 016）

## 🏗️ 技术设计 (Technical Design)

### 📐 架构概览
双击语音输入功能在现有Tencent Cloud ASR基础上进行交互层优化，保持底层ASR功能不变，仅修改用户触发逻辑：

1. **双击检测模块**: 检测300ms内的连续两次空格键按下
2. **状态管理模块**: 管理录音状态（开始/停止）
3. **用户反馈模块**: 提供清晰的操作提示和状态反馈

### ⚙️ 核心实现
#### 主要组件/模块
- **双击检测逻辑**: 在键盘事件处理器中实现300ms时间窗口的双击检测
- **录音状态切换**: 双击触发录音开始，再次双击或超时自动停止
- **用户提示优化**: 更新提示信息以匹配双击交互模式

#### 关键技术决策
- **300ms双击阈值**: 平衡误触和操作流畅性，符合常见双击交互标准
- **空格键作为触发键**: 空格键位置居中、易于按压，适合作为语音输入触发键
- **状态自动管理**: 录音开始后自动管理状态，用户无需关心技术细节
- **向后兼容**: 保留`/voice`命令作为备选方案

#### 数据流/状态管理
1. 用户双击空格键（300ms内两次按下）
2. 系统检测到双击事件，启动语音录制
3. 显示"🎙️ 开始语音输入... (再次双击结束录音)"提示
4. 用户说话，系统持续录制音频
5. 用户再次双击空格键或等待3秒无操作，停止录音
6. 将录制的音频发送到腾讯云ASR进行识别
7. 显示识别结果并作为用户输入处理

### 🧩 API 变更
#### 新增 API
无新增API，仅修改交互逻辑。

#### 修改的 API
- **src/index.ts**: 更新键盘事件处理逻辑，优化双击检测和用户提示
- **交互提示信息**: 从"松开 Cmd/Ctrl + R 结束"改为"再次双击结束录音"

#### 废弃的 API
- 移除了对Cmd/Ctrl+R组合键的依赖，简化为单一的双击空格键操作

## 🚀 使用指南 (Usage Guide)

### 📦 安装/配置
无需额外安装，此功能基于现有的Tencent Cloud ASR功能，只需确保已正确配置腾讯云认证信息。

### 🎮 基本使用
1. **启动AIBO交互模式**:
   ```bash
   npm start
   ```

2. **使用双击语音输入**:
   - 在命令行提示符下，**快速双击空格键**（300ms内两次）
   - 看到提示"🎙️ 开始语音输入... (再次双击结束录音)"
   - 开始说话
   - 说完后，**再次双击空格键**结束录音
   - 系统会显示识别结果并自动处理

3. **备选方案**:
   - 也可以使用`/voice`命令启动5秒固定时长的语音输入

### 🏆 高级用法
#### 自定义双击阈值
虽然当前硬编码为300ms，但未来可以考虑通过环境变量配置：
```env
VOICE_DOUBLE_CLICK_THRESHOLD=500  # 自定义双击时间阈值（毫秒）
```

#### 语音输入结果处理
识别结果会自动作为用户输入提交给AI Agent处理，支持所有常规命令和查询。

### 🔄 迁移指南
#### 从旧版本迁移
此功能为交互体验优化，无需代码迁移。用户只需了解新的双击操作方式即可。
