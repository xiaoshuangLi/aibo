# Enhanced System Prompt Structure and Workflow Optimization

## 📋 功能概述 (Feature Overview)

### 🎯 一句话描述
重构并优化了系统提示的结构和内容，采用更清晰的分层组织方式，同时优化了预提交工作流程和文档模板，提升了整体开发体验和代码质量保证。

### 💡 业务价值
- **解决的问题**: 原有的系统提示结构不够清晰，预提交钩子执行顺序不合理，feature 文档模板缺乏标准化
- **带来的价值**: 提供更清晰的 AI 行为指导，优化开发工作流程，确保更高的代码质量和测试覆盖率
- **目标用户**: 开发者、AI Agent 使用者、项目维护者

## 🏗️ 技术设计 (Technical Design)

### 📐 架构概览
本次变更主要涉及三个核心组件：
1. **系统提示模块** (`src/enhanced-system-prompt.ts`) - 重写为结构化格式
2. **预提交钩子** (`.husky/pre-commit`) - 优化执行顺序
3. **文档模板** (`templates/feature-template.md`) - 标准化 feature 文档格式

### ⚙️ 核心实现
#### 主要组件/模块
- **Enhanced System Prompt**: 采用分层结构，包含身份定义、环境上下文、核心能力、子代理框架、错误处理策略、操作规则、格式指南、问题解决方法论、功能开发工作流程等部分
- **Pre-commit Hook**: 调整检查顺序，先运行测试覆盖率检查，再检查覆盖率阈值，最后运行构建检查
- **Feature Template**: 提供完整的 feature 文档模板，包含功能概述、技术设计、使用指南、影响分析、验证要求、维护指南和变更日志

#### 关键技术决策
- **结构化系统提示**: 采用 Markdown 标题和 emoji 图标来增强可读性，使 AI 更容易理解和遵循指令
- **工作流优化**: 将测试覆盖率检查放在构建检查之前，确保只有通过测试的代码才能进入构建阶段
- **文档标准化**: 提供详细的 feature 文档模板，确保所有功能都有完整的文档记录

#### 数据流/状态管理
本次变更不涉及复杂的数据流或状态管理，主要是静态内容的重构和工作流程的优化。

### ✅ 必须验证的场景
1. **系统提示加载**: 确保 AI Agent 能正确加载新的系统提示

   - 验证方法: 启动 AI Agent 并观察其行为

2. **预提交钩子执行**: 确保预提交钩子按新顺序执行

## 🛠️ 维护指南 (Maintenance Guide)

### 🔍 调试技巧
- **常见问题 1**: 系统提示过长导致 token 限制
  - 解决方案: 优化提示内容，移除不必要的细节
- **常见问题 2**: 预提交钩子执行失败
  - 解决方案: 检查测试覆盖率是否达到 85% 以上，确保构建成功
- **日志级别**: 建议在调试时启用详细日志

### 📈 监控指标
- **关键指标 1**: 测试覆盖率（必须 ≥ 85%）
- **关键指标 2**: 构建成功率
- **告警阈值**: 测试覆盖率 < 85% 时触发告警

### 🔄 未来扩展
- **计划中的功能**: 添加更多语言支持
- **架构限制**: 系统提示长度受 token 限制
- **技术债务**: 需要为系统提示添加更好的测试策略
