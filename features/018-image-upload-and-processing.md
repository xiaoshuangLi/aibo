# 📸 第18号功能：图像上传与处理能力

## 功能概述
为 AIBO 添加完整的图像上传和处理能力，支持在终端和飞书模式下读取、分析和处理本地图像文件。

## 背景与动机
随着多模态 AI 模型的普及，视觉理解能力成为现代 AI 助手的重要组成部分。用户经常需要分析本地图像、截图或照片，但之前的 AIBO 版本缺乏直接处理图像文件的能力。此功能填补了这一空白，使 AIBO 能够：
- 读取用户指定的本地图像文件
- 将图像转换为 Base64 编码供视觉模型分析
- 在飞书集成中支持图像消息的发送和接收
- 为后续的图像识别、分析和生成功能奠定基础

## 核心组件

### 1. 图像读取工具 (`src/tools/read-image.ts`)
- **功能**：读取本地 JPEG、PNG、GIF、WebP 格式的图像文件
- **输出**：返回 Base64 编码的图像数据，格式为 `data:image/{format};base64,{encoded_data}`
- **集成**：自动与 vision-capable 模型（如 GPT-4o、Claude 3.5 Sonnet）集成
- **使用场景**：用户要求 "分析这张图片" 或指定图像文件路径时自动调用

### 2. 图像上传中间件 (`src/core/middlewares/image-upload.ts`)
- **功能**：处理图像文件上传逻辑，提供统一的图像处理接口
- **特性**：支持大文件分块处理，内存优化
- **集成**：作为核心中间件，可被任何需要图像处理的 Agent 调用

### 3. WebSocket 客户端 (`src/presentation/lark/ws-client.ts`)
- **功能**：实现实时双向通信，支持飞书机器人的高级交互功能
- **用途**：为图像上传、实时状态更新等场景提供底层通信支持

### 4. 飞书适配器增强
- **功能**：更新飞书适配器以支持图像消息的发送和处理
- **特性**：支持用户在飞书聊天中直接发送图像，AIBO 自动接收并分析

## 技术实现细节

### 文件格式支持
- **JPEG**：最常用的有损压缩格式
- **PNG**：支持透明度的无损压缩格式  
- **GIF**：支持动画和简单颜色的格式
- **WebP**：Google 开发的现代格式，提供更好的压缩率

### Base64 编码
图像文件被自动转换为标准的 Data URL 格式：
```
data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAYABgAAD...
data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...
```

### 内存管理
- 大图像文件采用流式读取，避免内存溢出
- 自动检测文件大小，对超大文件提供警告
- 临时文件自动清理机制

## 使用示例

### 终端模式
```bash
# 启动 AIBO
aibo

# 用户对话示例
用户: "请分析 images/screenshot.png 这张截图"
AIBO: (自动调用 read-image 工具读取文件)
AIBO: "这张截图显示了一个网页界面，包含..."
```

### 飞书模式
- 用户在飞书群聊中发送图像文件
- AIBO 自动接收图像并回复分析结果
- 支持基于图像内容的后续对话
