# 📸 第18号功能：图像上传与处理能力

## 📌 需求背景 (Requirements Background)

随着多模态 AI 模型的普及，视觉理解能力成为现代 AI 助手的重要组成部分。用户经常需要分析本地图像、截图或照片，但之前的 AIBO 版本缺乏直接处理图像文件的能力。此功能填补了这一空白，使 AIBO 能够读取用户指定的本地图像文件，将图像转换为 Base64 编码供视觉模型分析，并在飞书集成中支持图像消息的发送和接收。

## 📋 Specification (规格说明)

### 📑 功能变更清单
- **新增工具**: `read-image` 工具，支持读取本地图像文件
- **新增中间件**: `image-upload` 中间件，处理图像上传逻辑
- **新增模块**: `ws-client` WebSocket客户端，支持飞书实时通信
- **增强适配器**: 终端和飞书适配器支持图像处理
- **新增测试**: 完整的单元测试和集成测试覆盖
- **文档更新**: README.md、docs/env.md 和功能规格文档

### 功能范围
- 支持读取本地 JPEG、PNG、GIF、WebP 格式的图像文件
- 自动将图像转换为标准 Data URL 格式 (`data:image/{format};base64,{encoded_data}`)
- 在终端模式下，用户可以通过指定文件路径让 AIBO 分析图像
- 在飞书模式下，用户可以直接发送图像文件，AIBO 自动接收并分析
- 与现有的 vision-capable 模型（如 GPT-4o、Claude 3.5 Sonnet）无缝集成

### 技术约束
- 文件大小限制：默认 10MB（可配置）
- 支持的文件格式：JPEG、PNG、GIF、WebP
- 安全要求：防止目录遍历攻击，文件路径验证
- 内存管理：大文件采用流式读取，避免内存溢出

### 非功能性需求
- **性能**：图像处理响应时间 < 2秒（对于 <10MB 文件）
- **安全性**：完整的安全防护措施
- **兼容性**：完全向后兼容，不影响现有功能
- **可维护性**：模块化设计，易于扩展

## 🏗️ Technical Design (技术设计)

### 架构概述
采用分层架构设计，包含以下核心组件：

1. **图像读取工具层** (`src/tools/read-image.ts`)
   - 负责底层文件读取和格式检测
   - 实现 Base64 编码转换
   - 提供统一的工具接口

2. **中间件处理层** (`src/core/middlewares/image-upload.ts`)
   - 处理图像上传逻辑
   - 提供缓存机制避免重复上传
   - 作为核心中间件供所有 Agent 使用

3. **通信适配层** (`src/presentation/lark/ws-client.ts`)
   - 实现 WebSocket 实时通信
   - 支持主从进程模式
   - 提供飞书集成的底层通信支持

4. **表现层适配器**
   - 终端适配器：支持命令行图像分析
   - 飞书适配器：支持聊天中的图像消息处理

### 数据流设计
```
用户请求 → 工具调用 → 文件读取 → Base64编码 → 模型分析 → 结果返回
```

### 安全设计
- **输入验证**：文件路径白名单验证
- **大小限制**：10MB 默认限制，可配置
- **格式验证**：仅支持安全的图像格式
- **内存管理**：流式读取，临时文件自动清理

## 🚀 Usage Guide (使用指南)

### 终端模式使用

#### 基本用法
```bash
# 启动 AIBO
aibo

# 用户对话示例
用户: "请分析 images/screenshot.png 这张截图"
AIBO: (自动调用 read-image 工具读取文件)
AIBO: "这张截图显示了一个网页界面，包含..."
```

#### 支持的命令
- 直接指定文件路径：`"分析 /path/to/image.jpg"`
- 相对路径：`"描述 images/example.png"`
- 绝对路径：`"查看 /Users/name/Pictures/photo.jpeg"`

### 飞书模式使用

#### 基本流程
1. 在飞书群聊或私聊中@AIBO机器人
2. 直接发送图像文件（拖拽或选择文件）
3. AIBO 自动接收图像并回复分析结果
4. 可以基于图像内容进行后续对话

#### 支持的功能
- 图像内容描述
- 图像中的文字识别（OCR）
- 图像场景分析
- 基于图像的问答

### 环境配置
确保使用支持视觉能力的 AI 模型：
- **OpenAI**: gpt-4o, gpt-4-vision-preview
- **Anthropic**: claude-3-5-sonnet-20241022, claude-3-opus-20240229
- **Google**: gemini-1.5-pro, gemini-1.5-flash
